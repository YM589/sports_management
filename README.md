# 项目概述

本系统基于Django框架与MySQL数据库开发，面向田径运动会打造数字化管理平台，划分**管理员、裁判、运动员**三类角色，实现赛事管理、报名、成绩录入、排名核算、团队积分、公告发布等全流程线上化，简化赛事组织流程，提升数据管理效率与准确性。 



# 功能介绍 

## 多角色权限管理

- **管理员**：全生命周期赛事管理、用户账号创建与权限分配、数据报表导出、系统公告发布、赛事日志监控
- **裁判**：专属看板查看项目数据、成绩实时录入与有效性校验、自动排名更新、接收项目定向公告
- **运动员**：赛事项目一键报名（名额自动校验）、成绩与排名查看、团队积分实时追踪、弃权申请与原因提交

## 核心赛事管理

- 支持 100 米跑、跳远、铅球等多类型田径项目配置，可设置参赛人数上限、报名截止时间、赛程阶段
- 赛事状态自动流转（未开始 / 进行中 / 已结束），报名截止后自动关闭注册入口，未确认成绩禁止结束赛事
- 成绩录入支持浮点数，自动拒绝负数 / 非数字输入，确认后锁定成绩防止篡改，同步触发团队积分计算

## 数据处理与可视化

- 集成 openpyxl/pandas 实现参赛数据批量导入、成绩报表导出（Excel/CSV 格式）
- 通过 matplotlib 生成成绩分布直方图、团队积分折线图等统计图表，辅助赛事数据分析
- 基于缓存技术优化数据加载效率，裁判 / 运动员看板缓存 1 分钟有效期，高频数据快速访问

## 数据完整性与校验

- 数据库模型通过一对一 / 多对多 / 一对多关联构建业务逻辑，确保用户、项目、成绩、团队数据关联一致
- 自动化校验：报名名额校验、运动员 ID 合法性校验、成绩有效性校验、重复报名提示、团队删除前置校验
- 实时数据更新：成绩确认后自动更新排名、团队积分，看板数据同步刷新，清除缓存保证数据最新

## 其他实用功能

- 系统公告按优先级 + 发布时间排序展示，支持定向发布与激活 / 禁用管理
- 运动员可按项目筛选历史成绩，成绩状态可视化标注（已确认 / 待审核）
- 裁判可查看负责项目的报名人数、已确认成绩人数、弃权人数等核心统计数据


# 项目亮点
1. 防篡改机制：成绩确认后永久锁定，修改留痕，保证赛事数据公平公正
2. 高效数据处理：支持批量导入运动员信息、一键导出成绩报表，适配赛事。
3. 可视化分析：自动生成统计图表，辅助管理员快速掌握赛事整体情况
4. 健壮性校验：全流程数据校验，杜绝非法数据、重复报名、无效成绩


# 技术栈

- **开发语言**：Python 3.10
- **后端框架**：Django 5.1.3
- **数据库**：MySQL 8.0
- **工具库**：openpyxl、pandas、matplotlib
